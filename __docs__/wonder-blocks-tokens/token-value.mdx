# tokenValue()

Wonder Blocks tokens are exported as CSS `var(--...)` references so
that the active theme can switch values through the CSS cascade. This is great
for styling, but less useful when a third-party library expects a concrete
value such as `#1865f2` or `rgba(24, 101, 242, 0.5)`.

The `tokenValue` helper resolves a token to its computed raw value at runtime.

## Usage

```js
import {tokenValue} from "@khanacademy/wonder-blocks-tokens";

tokenValue(token: string, element?: Element): string;
```

## API

| Argument  | TypeScript Type | Default                    | Description                                                                                                                           |
| --------- | --------------- | -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| `token`   | `string`        | _Required_                 | A token value, typically a `var(--...)` reference from this package. Values that are not `var(...)` expressions are returned as-is.   |
| `element` | `Element`       | `document.documentElement` | The element whose computed style is read. Pass a descendant of a `[data-wb-theme]` element to resolve the value for that theme scope. |

The function returns the trimmed raw value. If the custom property is not
defined on the given element, an empty string is returned.

## Basic Example

This example resolves the same token twice — once in the `default` theme scope
and once inside a `thunderblocks` scope — to show how the returned value
reflects the active theme.

<Canvas of={TokenValueDefault} />

```tsx
import {semanticColor, tokenValue} from "@khanacademy/wonder-blocks-tokens";

// Reads the active theme's value from :root.
const instructive = tokenValue(
    semanticColor.core.foreground.instructive.default,
);

// Reads the value that would apply inside a specific theme scope.
const instructiveInScope = tokenValue(
    semanticColor.core.foreground.instructive.default,
    elementInsideThunderblocksScope,
);
```

## Overriding the `element` argument

The `element` argument is optional and defaults to the first `[data-wb-theme]`
element in the current document, falling back to `document.documentElement`.
Pass a descendant of a `[data-wb-theme]` element to resolve the value for that
theme scope.

<Canvas of={TokenValueElementOverride} />

```tsx
import {semanticColor, tokenValue} from "@khanacademy/wonder-blocks-tokens";

const instructive = tokenValue(
    semanticColor.core.foreground.instructive.default,
    elementInsideThunderblocksScope,
);
```

## Non-semantic tokens

The `tokenValue` helper can also be used to resolve the raw value of
non-semantic tokens.

<Canvas of={NonSemanticTokenValue} />

```tsx
import {sizing, tokenValue} from "@khanacademy/wonder-blocks-tokens";

const size = tokenValue(sizing.size_080);
```
