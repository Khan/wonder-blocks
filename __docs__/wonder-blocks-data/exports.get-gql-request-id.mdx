# getGqlRequestId()

```ts
function getGqlRequestId<TData, TVariables: {...}>(
    operation: GqlOperation<TData, TVariables>,
    variables: ?TVariables,
    context: GqlContext,
): string;
```

The `getGqlRequestId` function generates an identifier based on the operation, variables, and context of a specific GraphQL request. This identifier is guaranteed to be the same for all requests that share the same operation, variables, and context, even if variables and context values are in different orders.

The identifier returned by this function can then be used with our <a href="./?path=/docs/packages-data-exports-usecachedeffect--docs">`useCachedEffect`</a>, <a href="./?path=/docs/packages-data-exports-useservereffect--docs">`useServerEffect`</a>, and <a href="./?path=/docs/packages-data-exports-usehydratableeffect--docs">`useHydratableEffect`</a> hooks as the `requestId` parameter, allowing them to be combined with the fetch operation from the <a href="./?path=/docs/packages-data-exports-usegql--docs">`useGql`</a> hook.
