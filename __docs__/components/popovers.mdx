# Popovers

## Popovers

Contextual overlay/Rich information container

A non-modal overlay that displays rich content, including text, links, icons, and/or illustration, triggered by a user interaction with a specific UI element. Unlike tooltips, popovers are designed for larger bodies of info and interactive content.

<img src={popovers1Img} alt="Popovers 1" />

### When to use

The choice to use a popover is determined by the need to provide supplemental information or micro-tasks that are too complex for a tooltip but do not require the full interruption of a modal.

| Use when | Example use |
| --- | --- |
| Displaying formatted text, headers, or small snippets of info | Explaining a specific exercise, skill name, and score, e.g. "Familiar.” |
| The overlay needs to contain clickable links, buttons, or small forms | A popover that includes a "Jump here" button to navigate the user to a specific activity. |
| Delivering transient, proactive feedback that requires user acknowledgement | Khanductor appearing in a popover to say "Slow down just a bit" with a "Got it" button during an assessment. |
| A specific on-screen element needs contextual guidance or a nudge tied directly to it | Khanmigo anchored to the gem counter, prompting the student on where they can check progress toward earning a next accessory. |

### When NOT to use

| Don’t use when | Recommended alternative | Rationale |
| --- | --- | --- |
| The content requires extensive reading across multiple paragraphs | Modal | Do not use popovers for multiple paragraphs. If the content requires extensive reading, use a modal to focus the user's attention. |
| Only a simple, unformatted identification label is needed | Tooltip | A popover introduces a structured overlay with padding, elevation, and potential actions — all unnecessary overhead for a single identifying label. Tooltips are lightweight by design. |
| The user must complete a complex task or make a destructive choice | Modal | If the user must complete a complex task or make a destructive choice, use a modal to block the background interaction. |
| The message requires the user's full attention before proceeding | Modal | Popovers are anchored to a specific UI element and non-blocking — they can't command full attention the way a modal can |

### Variants

Each variant serves a distinct use case. Choose based on the intent of the content — what the user needs to do or understand when the popover appears.

### Informational

Focuses on providing additional context as succinctly as possible, such as exercise names and learner scores. Typically action-oriented, including a button for immediate navigation — e.g. "Jump here" — task completion, or more info to read.
There can be no more than two buttons — one primary and one tertiary.

<img src={popovers2Img} alt="Popovers 2" />

### Feature highlight

Used to introduce a new feature when the feature can be introduced in context, anchored to the relevant UI element, without interrupting the user's flow.
Prompted by the product. Used for single-step guidance only.

<img src={popovers3Img} alt="Popovers 3" />

### System guidance

Used for transient, proactive feedback from the system that requires user acknowledgement.
Always includes a primary acknowledgement action — e.g. "Got it" or "Okay." May include a secondary tertiary option.

<img src={popovers4Img} alt="Popovers 4" />

### Usage and purpose by variant

| Intent | Purpose | Use case |
| --- | --- | --- |
| Informational | To surface contextual detail about a specific UI element without leaving the page. | Explaining an exercise name, skill level, and score inline with the learning queue. |
| Feature highlight | Introduces a new feature in context, anchored to the relevant UI element, without interrupting the user's flow. | Onboarding a student to the gem counter and rewards system after first login. |
| System guidance | To deliver proactive feedback from the system that requires acknowledgement before continuing. | Khanmigo prompting "Slow down just a bit" during an assessment. |

### Placement

### Adaptive relative anchoring

Popovers must always anchor and appear from the content that triggered them. When they prefer "above" or "below" placement, they must flip to the opposite or reverse orientation if they would be clipped by the viewport edge.

<img src={popovers5Img} alt="Popovers 5" />

### States

### With character

Use a character illustration when the message is being delivered on behalf of a named system character — Khanmigo (tutor) or Khanductor (proctor). The character signals the source of the message and adds personality to system-initiated communication.
Do not use a character illustration if the message is not being delivered on behalf of Khanmigo or Khanductor. If neither character is the source, use text only or a feature image instead.
Available in: System guidance only. Not appropriate for Feature highlight or Informational variants.
