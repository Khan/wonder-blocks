# Wonder Blocks Tokens

Wonder Blocks uses a set of tokens to define the look and feel of our
components. These tokens are defined in the `@khanacademy/wonder-blocks-tokens`
package.

Design tokens are a set of key-value pairs that define the visual and behavioral
design attributes of our components. They are the primitives that define the
building blocks of our design system.

## Tokens

These represent the design decisions at Khan Academy.

### Primitives

- [Border](../?path=/docs/packages-tokens-border--docs)
- [Sizing](../?path=/docs/packages-tokens-sizing--docs)
- [Typography](../?path=/docs/packages-tokens-typography--docs)
- [Media Queries](../?path=/docs/packages-tokens-media-queries--docs)
- [Color (deprecated)](../?path=/docs/packages-tokens-color--docs)
- [Spacing (deprecated)](../?path=/docs/packages-tokens-spacing--docs)

### Semantic

- [Box Shadow](../?path=/docs/packages-tokens-box-shadow--docs)
- [Semantic Colors](../?path=/docs/packages-tokens-semantic-color--docs)
- [Deprecated Semantic Colors](../?path=/docs/packages-tokens-semantic-color--docs)

## Usage

To use these tokens, you can import them from the
`@khanacademy/wonder-blocks-tokens` package.

```ts
// to import all the tokens at once:
import * as tokens from "@khanacademy/wonder-blocks-tokens";
// or to import a specific token:
import {color} from "@khanacademy/wonder-blocks-tokens";
```
